2.- Convert To STL EBU Step-By-Step Guide

To convert an exl, srt, rtf, txt or STL EBU V2 subtitle file to an STL EBU subtitle file, we need to go through the following steps:
Write the metadata in area number 1 (STL EBU Metadata),
define the top and bottom vertical positions for subtitles in area 2 (Vertical position control),
select the source file in area 4 (Convert to STL EBU), and convert it to the STL EBU format.

Let’s take a closer look at each of these fields.

Area 1: STL EBU Metadata - Setup

It is okay to leave fields empty in this area, unless you wish to write in helpful information for your STL EBU file. This information won’t change your subtitles (for example, changing the “width” in the metadata won’t change the actual width in your subtitles), with the exception of the “Character code table (CCT)”, which tells the software the alphabet you are using to encode your subtitles. This version of the software only supports the Latin alphabet, but in future versions you will be able to select from the Cyrillic, Arabic, Greek and Hebrew alphabets.

Under “Original title:” write the original title of the content you are working on. This field only recognizes up to 32 characters; if you write any more, they will be ignored.

If you are working on a serial, write the episode title up to 32 characters long in the field under “Episode title”. The following fields, “Translated title” and “Translated episode title”, should contain the designated titles for the series/movie and/or episode, respectively, for the region you are working in. This is usually provided by the owner of the content in that particular area of the world. It is standard practice to write TITLE PENDING and EPISODE TITLE PENDING for these fields. This is why these titles appear by default.

Under “Language”, there is a drop-down menu where you can select the language you will be using. Three languages appear by default: English, Spanish and Portuguese. If you will be using a different language, pick “Other” and write in the code for that language in the field under “Code for other language”. The language code table is included in Chapter 4 of this guide.

Using the “Width” drop-down menu, select the maximum amount of characters per line, as set in your subtitling preparation.

Sometimes, a subtitle file needs to be changed. If you’ve changed your subtitle file, and thus, generated a new file in the metadata, change the number on the “Revision #” drop-down menu to let the software know that this is a newer version of the file.

There are two dedicated fields to write your name and contact (your e-mail address, for example). Both fields recognize up to 32 characters, so any more than that will be ignored.

The “Code Page Number” drop-down menu in this version only has one option, the “850 Multilingual” code page, which is very common. Its function is to tell other systems that the metadata has been encoded using this code page.

The “DFC” (which stands for Disk Format Code) drop-down menu allows you to select the following television frame rates: STL25.01 for 25 fps (which is the European format, known as PAL), and STL30.01 for 30 fps (which is the North and South American format, known as NTSC). These are the two frame rates included in the official STL EBU definition. However, we’ve added a third, the STL24.01 for 24fps, which is quite common nowadays.

The “Character code table” drop-down menu for this version only has one option, but it is intended for you to choose the alphabet you will be working with. This is, the letters and symbols you can use to generate a proper STL EBU file. Not every letter is accepted in the STL EBU format, so this means that all letters and symbols used in your subtitling must be included in the STL EBU table for the alphabet you chose. If you use a letter or symbol outside the STL EBU format definition, the software will display a warning so you can change it and try the conversion again.

Under “Creation date”, select the date in which your subtitling was created. The date set on your computer is displayed by default.

Area 2: Vertical Position Control - Setup

Vertical position in the STL EBU format is defined in two ways: as in-vision with values ranging from 0-99,
or as teletext with values ranging from 1-12. This number represents the number of rows from the top of the screen
to the bottom subtitle row. Subtitle to STL EBU uses the most common of the two, which is teletext, reads its vertical positions,
and translates them to the two vertical positions that the software supports for layback: TOP and BOTTOM.

In your subtitling preparation, define where your subtitles will be displayed. If you want them to appear at the top of the screen, “raise” them by adding a bracket “}” at the beginning of each subtitle. The default position for a subtitle is the bottom of the screen, so there is no need to add anything if you want the subtitle to be displayed here.

In the Vertical Position Control area you have two areas, one labeled as “To STL EBU” and the other “From STL EBU”. In this section we will go through the “To STL EBU” area and we will set up the vertical position for subtitles created for the BOTTOM or TOP areas.

Since STL EBU has 12 vertical positions, “1” represents the top of the screen, “6” the middle,
and “11” the bottom of the screen (line number 12 being the second subtitle line).

In the “Raised line” drop-down menu, select the row where you want the “raised” subtitles to be displayed. The default value is 02, which is the industry standard.

In the “Base line” drop-down menu, select the row where you want the bottom subtitles to be displayed. The default value is 11, which is the industry standard.

We are now going to jump to area four, where we will go through the selection of our source subtitle file and getting the STL EBU we want. We will go back to area 3 further down this guide.

Area 4: Convert To STL EBU - Setup

Press the button from the format you wish to convert to an STL EBU file. For example, if you have an exl subfile, you should press the EXL to STL button. Same thing for the RTF to STL EBU or the TXT to STL EBU (TXT must be UTF-8 encoded). When you choose SRT to STL EBU, you should also select the corresponding text encoding: UTF8, WIN or MAC. STL EBU V2 to STL EBU refers to a non-official version of the STL EBU which lacks metadata and is used mainly for distribution purposes. The button works in the same way.

Click on the button and a browser window asking you to open a subtitle file will be displayed. Once you’ve opened your subfile, a new browser window will open asking you to choose a name and location for the new subtitle file you want to save. A default name is provided, but you can overwrite it. When you’re finished, simply press “Save”.

Optionally at top of this area, you will find three check buttons:

”Save report in a rtf file”. When checked, a report will be generated and saved to the disk. The report includes a list of subtitles with italics, “raiseds” and forceds. It also includes information on the biggest frame in your file (so you know the frame rate at which your file was prepared), the total number of subtitles converted, and the actual maximum width of your subtitles.
“Keep raised sign in the text field”. The STL EBU format has a dedicated area inside the data to store the vertical position of the current subtitle, so the flag “}” used in text-based formats is no longer needed and can be erased from the actual text of the subtitle. Despite this, most subtitle labs will ask you to keep this flag in the text. For that purpose, check this option.
“Italics in text field only”. Subtitles to STL EBU will change the italics text flags into data, erasing these brackets from the text field. Nowadays, though, most subtitling labs will require this option to be checked so that the italics flags will be kept in the subtitle text and there won’t be any italics in the STL EBU data.

Examples of these formats are included in Chapter 4 of this guide.
