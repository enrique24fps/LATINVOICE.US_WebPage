3.- Convert From STL EBU Step-By-Step Guide.

To convert an STL EBU subtitle file to an EXL, SRT, RTF, TXT or STL EBU V2 subtitle file, we need to go through the following steps:

define the top and bottom vertical positions for subtitles in area 2 (Vertical position control),
select the source file in area 5 (Convert from STL EBU), and convert it to the EXL, SRT, RTF, TXT or STL EBU V2 subtitle format.

This type of conversion won’t use the area one (STL EBU Metadata) because the STL EBU we are about to convert has its own metadata embedded.

Let’s take a closer look at each of these fields.

Area 2: Vertical Position Control - Setup

Vertical position in the STL EBU format is defined in two ways: as in-vision with values ranging from 0-99,
or as teletext with values ranging from 1-12. This number represents the number of rows from the top of the screen
to the bottom subtitle row. Subtitle to STL EBU uses the most common of the two, which is teletext, reads its vertical positions,
and translates them to the two vertical positions, (TOP and BOTTOM) that are widely compatible with most subtitle workflows.

Since STL EBU has 12 vertical positions, “1” stands for the top of the screen, “6” for the middle,
and “11” for the bottom (since there are two lines in each subtitle, i.e. 11 and 12).
Thus, when you choose “6” in the drop-down menu, it means that any subtitle with a vertical position equal or less than 6
will appear at the TOP in your new subtitle file, and any vertical position bigger than 6 will appear at the BOTTOM in your new subtitle file.

In the “Raiseds are at line … or below” drop-down menu, select the starting row from which subtitles will be “raised”. Once converted, those subtitles will begin with the symbol “}”, and they will be flagged to be displayed at the top of the screen. The default value is 06.

Note that the SRT format by definition does not support “raiseds”.

Area 5: Convert From STL EBU - Setup

Press the button to the format you wish to convert an STL EBU file. For example, if you want to generate an EXL subfile from an STL EBU file, you should press the “STL EBU to EXL” button. Same thing with the “STL EBU to RTF” or the “STL EBU to TXT” (TXT must be UTF-8 encoded). Press “SRT to STL EBU” and you will get an UTF8-encoded SRT file. STL EBU V2 to STL EBU refers to a non-official version of the STL EBU which lacks metadata and is used mainly for distribution purposes. Button works in the same way.

Click on the button and a browser window asking you to “open” your STL EBU subtitle file will be displayed. Once you’ve opened your subfile, a new browser window will open asking you to choose a name and location for the new subtitle file you want to save. A default name is provided, though you can overwrite it. When you’re finished, simply press “Save”.

Optionally at top of this area, you will find two check buttons:

”Save metadata in a rtf file”. When checked, a report will be generated in a separate file with the metadata embedded in the STL EBU file.
“Consolidate double raiseds”. The STL EBU format has a dedicated area in the data to store the vertical position of the current subtitle. If a particular subtitle happens to have the “raised” information duplicated -in the data as well as in the text area-, the software will display a double flag “}}” in the target subtitle file. When checking this option, all double flags will be changed to a single flag “}”.

Examples of this formats are included in Chapter 4 of this guide.
